### Реализация

#### Класс `TaskQueue`

Коллекция задач, основанная на словаре `dict`. Поддерживает протокол итерации и ленивую фильтрацию.

```python
class TaskQueue:
    def __init__(self):
        self.task_dict = {}
        self.last_num = 1

    def __iter__(self):
        for task in self.task_dict.values():
            yield task

    def add_task(self, task: Task):
        self.task_dict[self.last_num] = task
        self.last_num += 1

    def filter(self, *, status: str = None, high_priority: bool = None):
        ...
```

#### Протокол итерации

Метод `__iter__` реализован через генератор (`yield`), что обеспечивает:
- Ленивое получение элементов
- Поддержку повторного обхода (каждый вызов `iter()` создает новый генератор)
- Автоматическую обработку `StopIteration`

#### Ленивые фильтры

Метод `filter()` принимает ровно один именованный параметр:

| Параметр | Тип | Описание |
|---|---|---|
| `status` | `str` | Фильтр по статусу задачи |
| `high_priority` | `bool` | Задачи с приоритетом >= 3 |
| `priority` | `int` | Фильр по указанному приоритету |

Реализован через `yield` — возвращает генератор, не загружая все данные в память.

### Демонстрация

#### 1. Итерация по очереди

```python
for task in queue:
    print(task.description)
```

#### 2. Повторный обход

```python
first_pass = [task.id for task in queue]
second_pass = [task.id for task in queue]
# first_pass == second_pass -> True
```

#### 3. Обработка `StopIteration`

```python
iterator = iter(queue)
try:
    while True:
        task = next(iterator)
except StopIteration:
    print("Queue exhausted")
```

Генератор автоматически бросает `StopIteration` по завершении, цикл `for` перехватывает его неявно.

#### 4. Ленивые фильтры

```python
# Фильтр по статусу
for task in queue.filter(status="In_progress"):
    print(task.description)

# Фильтр по высокому приоритету
for task in queue.filter(high_priority=True):
    print(task.description)
```

#### 5. Совместимость со стандартными конструкциями Python

```python
all_tasks = list(queue)  # list()
total = sum(task.priority for task in queue)  # sum()
count = sum(1 for task in queue if task.is_ready)  # for + условие
```

#### 6. Валидация фильтров

```python
queue.filter()                                  # ValueError: Choose one filter
queue.filter(status="Created", high_priority=True)  # ValueError: Choose one filter
```

### Выводы

В ходе выполнения лабораторной работы были изучены и применены следующие концепции:

1. **Протокол итерации** — реализован метод `__iter__` через генераторную функцию. Это позволяет использовать `TaskQueue` в любых стандартных конструкциях Python (`for`, `list()`, `sum()`) без дополнительной реализации `__next__`.

2. **Генераторы** — использование `yield` вместо накопления списков позволяет обрабатывать данные лениво. Метод `filter()` возвращает генератор, который вычисляет следующий элемент только при запросе, что эффективно при работе с большими коллекциями.

3. **Повторный обход** — поскольку `__iter__` является генераторной функцией (а не генераторным объектом), каждый вызов `iter(queue)` создает новый независимый итератор. Это обеспечивает корректный повторный обход.

4. **`StopIteration`** — генератор автоматически бросает исключение при завершении. Цикл `for` перехватывает его неявно, а при использовании `next()` вручную требуется явная обработка через `try/except`.

5. **Отсутствие избыточного хранения данных** — фильтрация не создает промежуточных списков, данные передаются напрямую из исходного словаря через цепочку генераторов.

6. **Валидация параметров** — метод `filter()` принудительно требует ровно один фильтр, что предотвращает некорректное использование и упрощает логику метода.
